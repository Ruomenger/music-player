# 国际化 (i18n) 设计

> **状态：Phase 8 已落地。** `LanguageManager` (src/app/) 装载 QTranslator；
> `resources/translations/musicplayer_zh_CN.ts` 由 `qt6_add_translations`
> (src/CMakeLists.txt) 在构建期编为 .qm 嵌入到可执行；MainWindow 与
> SettingsDialog 走 `retranslateUi()` 模式动态刷新。其它 UI 控件的
> retranslate 钩子在做增量补 —— 见末尾 "未完成项"。

## 概述

使用 Qt Linguist 工具链实现中/英文界面切换，支持运行时动态切换。

## 翻译流程

```mermaid
源代码 (tr()) → lupdate → .ts 文件 → Qt Linguist 翻译 → lrelease → .qm 文件 → 运行时加载
```

## 源代码中使用 tr()

```cpp
// 所有用户可见字符串用 tr() 包裹
class MainWindow : public QMainWindow {
    Q_OBJECT
public:
    MainWindow() {
        setWindowTitle(tr("Music Player"));
        
        auto fileMenu = menuBar()->addMenu(tr("&File"));
        auto openAction = fileMenu->addAction(tr("&Open File..."));
        
        playBtn_->setToolTip(tr("Play"));
        pauseBtn_->setToolTip(tr("Pause"));
    }
};
```

## CMake 集成

实际 `src/CMakeLists.txt` 在 `add_executable(musicplayer ...)` 之后调一次
`qt6_add_translations`（Qt 6.7+ 提供，需 `find_package(Qt6 ... LinguistTools)`）：

```cmake
qt6_add_translations(musicplayer
    TS_FILES
        ${CMAKE_SOURCE_DIR}/resources/translations/musicplayer_zh_CN.ts
    RESOURCE_PREFIX "/translations"
    LUPDATE_OPTIONS "-no-obsolete"
)
```

行为：

- `lrelease` 在 build 期自动跑，产出 `musicplayer_zh_CN.qm`
- 自动生成一个 .qrc 把 .qm 文件嵌进 musicplayer 可执行；运行时加载路径就是
  `:/translations/musicplayer_zh_CN.qm`
- `RESOURCE_PREFIX` 与 `QM_FILES_OUTPUT_VARIABLE` 互斥 —— 用前者把 .qm 自动
  打进资源，用后者拿到文件名自己处理
- `-no-obsolete` 让 lupdate 删除已被移除的源字符串；不加的话过时条目会标成
  `<obsolete>` 但仍留在 .ts 里
- lupdate 不会自动跑：手动 `cmake --build build --target musicplayer_lupdate`
  扫描源码、把新的 tr() 写进 .ts

## 翻译文件管理

```
resources/translations/
└── musicplayer_zh_CN.ts    # 中文翻译源文件 (XML)
```

> **不维护 `musicplayer_en.ts`**：英文是 `tr()` 调用里的源字符串，未安装翻译时 Qt 直接返回原字符串。再多一个空 `en.ts` 只会让 `lupdate` 多一份待维护文件。如果将来要做美式 vs 英式英语区分，再加。

### .ts 文件示例

```xml
<?xml version="1.0" encoding="utf-8"?>
<!DOCTYPE TS>
<TS version="2.1" language="zh_CN">
<context>
    <name>MainWindow</name>
    <message>
        <location filename="../src/ui/main_window.cpp" line="25"/>
        <source>Music Player</source>
        <translation>音乐播放器</translation>
    </message>
    <message>
        <location filename="../src/ui/main_window.cpp" line="42"/>
        <source>&amp;File</source>
        <translation>文件(&amp;F)</translation>
    </message>
</context>
</TS>
```

## 运行时切换语言

实际实现没用单例 —— LanguageManager 由 `main()` 在 QApplication 之后、
构造任何 widget 之前 new 出来，作为非拥有指针注入 MainWindow / SettingsDialog。
这样测试时可以独立 new 一份不影响进程全局状态。

```cpp
// src/app/language_manager.h
class LanguageManager : public QObject {
    Q_OBJECT
public:
    explicit LanguageManager(QObject* parent = nullptr);

    // 返回 true 表示加载成功 (或 "en" 卸载成功); false 表示 .qm 找不到。
    bool setLanguage(const QString& locale);

    [[nodiscard]] QString currentLanguage() const;
    [[nodiscard]] static QStringList availableLanguages();  // {"en", "zh_CN"}

signals:
    void languageChanged(const QString& locale);

private:
    QTranslator translator_;
    QString current_ = QStringLiteral("en");
    bool installed_ = false;
};
```

要点：

- **"en" 是源语言**，不存在 musicplayer_en.qm；切到 "en" 时只是
  `removeTranslator`，让 `tr()` 走 fallback 返回字面量
- **同一 locale 重复调用是 no-op**：不重新 install，也不发 signal
- **跨实例换 QTranslator 受限**：QTranslator 不可拷贝/移动，所以
  setLanguage 复用同一个成员 `translator_` 调 `load(qmPath)` 把内容替换
  掉，而不是构造新的 QTranslator

### 动态更新 UI

Qt 在 `installTranslator` / `removeTranslator` 后会向所有 widget 投递 `QEvent::LanguageChange`。每个有动态文本的 widget 都需要 override `changeEvent` 并重新调用 `setText` / `setWindowTitle` 等：

```cpp
class MainWindow : public QMainWindow {
protected:
    void changeEvent(QEvent* e) override {
        if (e->type() == QEvent::LanguageChange) {
            retranslateUi();
        }
        QMainWindow::changeEvent(e);
    }

private:
    void retranslateUi() {
        setWindowTitle(tr("Music Player"));
        fileMenu_->setTitle(tr("&File"));
        openAction_->setText(tr("&Open File..."));
        playBtn_->setToolTip(tr("Play"));
        // ... 所有 tr() 字符串都在这里重新调用
    }
};
```

> **关键**：纯代码构建的 widget (本项目不用 .ui 文件) **没有自动 retranslateUi**。每个窗口/控件类必须自己 override `changeEvent` 并把所有 `tr()` 调用集中到 `retranslateUi()`，构造函数里也调用一次 `retranslateUi()` 初始化文字。

## 注意事项

1. **tr() 只在 QObject 子类中使用**, 非 QObject 类用 `QCoreApplication::translate("context", "text")`
2. **格式化字符串**: `tr("Playing: %1 - %2").arg(title).arg(artist)`
3. **复数处理**: `tr("%n songs", "", count)` (需要 Qt 版本支持)
4. **翻译文件更新**: 添加新字符串后跑 `cmake --build build --target musicplayer_lupdate` (qt6_add_translations 自动注册的 target)
5. **中文翻译**使用 Qt Linguist 工具 (GUI) 或直接编辑 XML 形式的 .ts 文件
6. **英文作为回退语言**: 代码中所有 `tr()` 的原始字符串即为英文, 英文 .ts 文件可为空

## 未完成项 (Phase 8 增量)

Phase 8 把 i18n 主路径打通了 (LanguageManager + zh_CN.ts + 嵌入 qrc)，但
"动态刷新所有 widget" 还没补完。下面这些控件目前只有构造时 `tr()` 一次，
切语言不会立即更新文字（重启后正常）：

- `ControlBar` — 模式按钮文字 / Volume tooltip
- `PlaylistWidget` — 右键菜单 "Play / Toggle favorite / Remove from playlist"
- `PlaylistSidebar` — section header "Sources" / "Playlists"，右键 "Rename… / Delete"，"+ New Playlist" 按钮
- `NewPlaylistDialog` — 窗口标题、Name/Description 标签、placeholder 文字
- `LyricWidget` — "Load lyrics…" 按钮、空状态 "(no lyrics)"、文件对话框
- `CoverWidget` — 无 tr() 内容，但 sizeHint 可能受字体度量影响

补法是常规套路：每个类加 `protected void changeEvent(QEvent*) override` 处理
`QEvent::LanguageChange`，把构造里的 `setText(tr(...))` / `setWindowTitle(tr(...))`
搬进一个 `retranslateUi()` 私函数并在构造时也调一次。MainWindow 和
SettingsDialog 已经走通这套，可作参考。
