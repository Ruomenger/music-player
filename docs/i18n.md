# 国际化 (i18n) 设计

> **状态：Phase 8 设计稿，未实现。** 代码侧目前没有 `LanguageManager`、
> `resources/translations/` 目录、`qt6_add_translations` 调用，UI 也没有
> `retranslateUi()` 钩子。本文档是 Phase 8 启动时的施工蓝图。

## 概述

使用 Qt Linguist 工具链实现中/英文界面切换，支持运行时动态切换。

## 翻译流程

```mermaid
源代码 (tr()) → lupdate → .ts 文件 → Qt Linguist 翻译 → lrelease → .qm 文件 → 运行时加载
```

## 源代码中使用 tr()

```cpp
// 所有用户可见字符串用 tr() 包裹
class MainWindow : public QMainWindow {
    Q_OBJECT
public:
    MainWindow() {
        setWindowTitle(tr("Music Player"));
        
        auto fileMenu = menuBar()->addMenu(tr("&File"));
        auto openAction = fileMenu->addAction(tr("&Open File..."));
        
        playBtn_->setToolTip(tr("Play"));
        pauseBtn_->setToolTip(tr("Pause"));
    }
};
```

## CMake 集成

```cmake
# 在 src/CMakeLists.txt 中
set(TS_FILES
    ${CMAKE_SOURCE_DIR}/resources/translations/musicplayer_zh_CN.ts
)

# lupdate 自定义目标 (扫描源码中的 tr())
add_custom_target(update_translations
    COMMAND ${Qt6_LUPDATE_EXECUTABLE}
        -recursive ${CMAKE_SOURCE_DIR}/src
        -ts ${TS_FILES}
    COMMENT "Updating translation files"
)

# 将 .qm 文件加入 Qt 资源
qt6_add_translations(musicplayer
    TS_FILES ${TS_FILES}
    QM_FILES_OUTPUT_VARIABLE QM_FILES
)
```

## 翻译文件管理

```
resources/translations/
└── musicplayer_zh_CN.ts    # 中文翻译源文件 (XML)
```

> **不维护 `musicplayer_en.ts`**：英文是 `tr()` 调用里的源字符串，未安装翻译时 Qt 直接返回原字符串。再多一个空 `en.ts` 只会让 `lupdate` 多一份待维护文件。如果将来要做美式 vs 英式英语区分，再加。

### .ts 文件示例

```xml
<?xml version="1.0" encoding="utf-8"?>
<!DOCTYPE TS>
<TS version="2.1" language="zh_CN">
<context>
    <name>MainWindow</name>
    <message>
        <location filename="../src/ui/main_window.cpp" line="25"/>
        <source>Music Player</source>
        <translation>音乐播放器</translation>
    </message>
    <message>
        <location filename="../src/ui/main_window.cpp" line="42"/>
        <source>&amp;File</source>
        <translation>文件(&amp;F)</translation>
    </message>
</context>
</TS>
```

## 运行时切换语言

```cpp
class LanguageManager : public QObject {
    Q_OBJECT
public:
    static LanguageManager& instance();

    void setLanguage(const QString& locale);  // "zh_CN" or "en"
    QString currentLanguage() const;
    
    QStringList availableLanguages() const {
        return {"zh_CN", "en"};
    }

signals:
    void languageChanged(const QString& locale);

private:
    QTranslator translator_;
    QString currentLang_;
};

void LanguageManager::setLanguage(const QString& locale) {
    qApp->removeTranslator(&translator_);
    
    QString qmFile = QString(":/translations/musicplayer_%1.qm")
                     .arg(locale);
    if (translator_.load(qmFile)) {
        qApp->installTranslator(&translator_);
        currentLang_ = locale;
        emit languageChanged(locale);
    }
}
```

### 动态更新 UI

Qt 在 `installTranslator` / `removeTranslator` 后会向所有 widget 投递 `QEvent::LanguageChange`。每个有动态文本的 widget 都需要 override `changeEvent` 并重新调用 `setText` / `setWindowTitle` 等：

```cpp
class MainWindow : public QMainWindow {
protected:
    void changeEvent(QEvent* e) override {
        if (e->type() == QEvent::LanguageChange) {
            retranslateUi();
        }
        QMainWindow::changeEvent(e);
    }

private:
    void retranslateUi() {
        setWindowTitle(tr("Music Player"));
        fileMenu_->setTitle(tr("&File"));
        openAction_->setText(tr("&Open File..."));
        playBtn_->setToolTip(tr("Play"));
        // ... 所有 tr() 字符串都在这里重新调用
    }
};
```

> **关键**：纯代码构建的 widget (本项目不用 .ui 文件) **没有自动 retranslateUi**。每个窗口/控件类必须自己 override `changeEvent` 并把所有 `tr()` 调用集中到 `retranslateUi()`，构造函数里也调用一次 `retranslateUi()` 初始化文字。

## 注意事项

1. **tr() 只在 QObject 子类中使用**, 非 QObject 类用 `QCoreApplication::translate("context", "text")`
2. **格式化字符串**: `tr("Playing: %1 - %2").arg(title).arg(artist)`
3. **复数处理**: `tr("%n songs", "", count)` (需要 Qt 版本支持)
4. **翻译文件更新**: 添加新字符串后运行 `cmake --build build --target update_translations`
5. **中文翻译**使用 Qt Linguist 工具 (GUI) 或直接编辑 XML 形式的 .ts 文件
6. **英文作为回退语言**: 代码中所有 `tr()` 的原始字符串即为英文, 英文 .ts 文件可为空
